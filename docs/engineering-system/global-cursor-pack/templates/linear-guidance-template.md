# Linear guidance (template for team)

Copy or adapt for your team. See also docs/engineering-system/linear-workflow.md.

- **When starting work:** Move issue to In Progress; create branch with issue ID (e.g. fix/PROJ-123-name).
- **In commits/PR:** Reference issue in message and PR body.
- **When merging:** Move issue to Done (or team equivalent).
- **Exploratory work:** No issue required; when formalizing, create issue and link branch/PR.
- **Labels/statuses:** Use existing team labels; use status to reflect reality (e.g. In Progress when coding).
