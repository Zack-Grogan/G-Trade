# Rules starter pack

Copy the .cursor/rules from a repo that uses this operating layer (e.g. G-Trade) or create from these names:

- 00-core-operating-contract.mdc — Read AGENTS.md first; structural changes follow contract.
- 05-development-stage-defaults.mdc — Issue-driven vs exploratory; do not fake issues.
- 10-docs-engine.mdc — Authored docs authority; generated docs refreshable; update docs when behavior changes.
- 20-architecture-reading.mdc — Read architecture before touching structure.
- 30-testing-discipline.mdc — Run tests; do not remove without reason.
- 40-issue-branch-pr-discipline.mdc — Reference issue in branch/commit/PR when present.
- 50-safety-and-non-destructive-behavior.mdc — No secrets; safe defaults; flag sensitive files.
- 60-openviking-knowledge-usage.mdc — Prefer repo docs; use OpenViking when helpful.
- 70-railway-and-infra-guardrails.mdc — (If using Railway) Deploy only to Railway; conservative usage.
- 80-reusable-template-boundary.mdc — Keep templates reusable across repos.

Adjust any repo-specific paths (e.g. plan file name) inside the rules.
