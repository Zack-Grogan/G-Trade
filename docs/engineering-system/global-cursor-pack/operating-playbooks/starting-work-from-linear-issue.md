# Starting work from a Linear issue

1. Read the issue (description, acceptance criteria, labels).
2. Read AGENTS.md and relevant docs (architecture, module affected).
3. Create a branch that references the issue (e.g. `fix/PROJ-123-short-name`).
4. Implement narrowly; run relevant tests; update docs if behavior or config changed.
5. Commit with issue ref in message (e.g. "Fix retry logic PROJ-123").
6. Open PR using template; link issue; summarize changes and testing.
7. Mark issue In Progress when starting; move to Done when merged (per team process).
