# Operating playbooks

Reusable step-by-step playbooks. Use as reference; adapt to your repo and tooling.

- **starting-work-from-linear-issue.md** — Read issue → branch → implement → verify → docs → PR.
- **exploratory-development.md** — No issue yet; implement narrowly → document → propose issue.
- **converting-discoveries-to-issues.md** — When to create an issue and how to link branch/PR.
- **creating-a-branch.md** — Naming (issue ref if any), from which base.
- **implementing-safely.md** — Plan narrowly, run tests, update docs, flag sensitive edits.
- **updating-docs.md** — When and what to update (architecture, operator, runbook, compliance).
- **opening-a-pr.md** — Use template, link issue, summarize changes and testing.
- **validating-before-merge.md** — Tests, lint, docs check, human review when needed.
- **refreshing-generated-docs.md** — Run generate_docs_index script; commit or CI.
- **refreshing-openviking.md** — When and how to re-ingest (if using OpenViking).
- **incident-or-debugging.md** — Collect context, locate modules, document findings, recommend issue.
- **deciding-local-vs-global.md** — Use local-vs-global checklist; put secrets and auth in global only.
